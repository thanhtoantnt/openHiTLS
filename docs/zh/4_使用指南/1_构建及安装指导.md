# 构建及安装指导

## 1. 准备构建环境

检查系统中构建工具是否已安装，并能正常使用。

| **名称** | **推荐版本** | **说明** |
| -------- | ------------ | -------- |
| Gcc        | >=7.3.0       | Linux    |
| CMake    | >=3.16         | Linux    |
| Sctp        | 无版本限制    | Linux    |

注：DTLS特性依赖sctp，默认sctp是关闭，如需开启，需要提前预装sctp依赖。
## 2. 准备源码

方式一

1. 下载openHiTLS代码，含业务代码、构建脚本、测试代码
   仓库地址：https://gitcode.com/openhitls/openhitls.git
2. openHiTLS依赖于libboundscheck库，构建之前需将其下载至openHiTLS/platform/Secure_C
   仓库地址：https://gitee.com/openeuler/libboundscheck.git

方式二

使用git submodule的方式下载，可以直接下载源码和依赖的Securec库，下载命令

```
git clone --recurse-submodules https://gitcode.com/openhitls/openhitls.git
```

## 3. openHiTLS构建及安装

进入openHiTLS后代码目录结构如下：

```
└── openHiTLS
   ├── bsl
   ├── CMakeLists.txt
   ├── cmake
   ├── config
   ├── crypto
   ├── docs
   ├── include
   ├── LICENSE
   ├── platform
   ├── README-zh.md
   ├── README.md
   ├── testcode
   ├── tls
   ├── pki
   └── auth
```

其中：

- bsl：存放基础功能相关代码；
- CMakeLists.txt：构建入口文件；
- cmake：存放 CMake 构建模块、特性选项定义、平台预设以及工具链文件等；
- config：存放构建时的配置头文件；
- crypto：存放密码学算法能力相关代码；
- platform：存放其他依赖的代码；
- testcode：存放测试工程类代码；
- tls：存放安全传输相关代码；
- pki：存放PKI相关代码；
- auth：存放认证相关代码；

**源码构建直接调用 CMake 进行构建，具体方法下面介绍。**

### 3.1 CMake构建

openHiTLS 使用纯 CMake 构建系统，通过 `-D` 参数直接控制所有构建选项。建议新建 build 目录用于存放构建过程中产生的临时文件，进入 build 目录后执行 `cmake .. && make` 完成构建。

CMake 构建的总体执行步骤如下：

```bash
cd openHiTLS
mkdir -p ./build
cd ./build
cmake .. [选项]  # 配置，详见3.1.1节
make -j
```

构建结果会输出在 openHiTLS/build 目录下。

常用 CMake 参数如下：

| **CMake 参数** | **参数说明** | **示例** |
| ------------- | ------------ | -------- |
| `HITLS_BUILD_PROFILE` | 使用预设配置，可选值：`full`、`iso19790` | `cmake .. -DHITLS_BUILD_PROFILE=full` |
| `HITLS_BSL` / `HITLS_CRYPTO` / `HITLS_TLS` / `HITLS_PKI` / `HITLS_AUTH` | 启用或禁用对应组件（ON/OFF） | `cmake .. -DHITLS_TLS=OFF` |
| `HITLS_CRYPTO_<算法名>` | 启用或禁用特定算法特性，请参考[特性说明](./4_配置指南.md#1-特性说明) | `cmake .. -DHITLS_CRYPTO_SHA256=ON` |
| `HITLS_BUILD_STATIC` | 构建静态库（默认 ON） | `cmake .. -DHITLS_BUILD_STATIC=ON` |
| `HITLS_BUILD_SHARED` | 构建共享库（默认 ON） | `cmake .. -DHITLS_BUILD_SHARED=ON` |
| `HITLS_BUNDLE_LIB` | 将所有模块打包为单个库 | `cmake .. -DHITLS_BUNDLE_LIB=ON` |
| `HITLS_BUILD_EXE` | 构建可执行命令行工具 | `cmake .. -DHITLS_BUILD_EXE=ON` |
| `HITLS_ASM` | 自动检测平台并启用汇编优化（x86_64 / ARMv8 / ARMv7） | `cmake .. -DHITLS_ASM=ON` |
| `HITLS_ASM_X8664` | 启用 x86_64 汇编优化 | `cmake .. -DHITLS_ASM_X8664=ON` |
| `HITLS_ASM_ARMV8` | 启用 ARMv8 汇编优化 | `cmake .. -DHITLS_ASM_ARMV8=ON` |
| `HITLS_ASM_ARMV7` | 启用 ARMv7 汇编优化 | `cmake .. -DHITLS_ASM_ARMV7=ON` |
| `HITLS_ASM_X8664_AVX512` | 启用 x86_64 AVX512 汇编优化。启用后构建系统会自动开启 `HITLS_ASM_X8664`，作为尚未实现 AVX512 优化的特性的默认优化 | `cmake .. -DHITLS_ASM_X8664_AVX512=ON` |
| `HITLS_COMPILE_OPTIONS` | 完全覆盖编译选项（分号分隔的 CMake 列表） | `cmake .. -DHITLS_COMPILE_OPTIONS="-O0;-g"` |
| `HITLS_SHARED_LINK_FLAGS` | 完全覆盖共享库链接选项（分号分隔的 CMake 列表） | `cmake .. -DHITLS_SHARED_LINK_FLAGS="-shared;-Wl,-z,now"` |
| `HITLS_EXE_LINK_FLAGS` | 完全覆盖可执行文件链接选项（分号分隔的 CMake 列表） | `cmake .. -DHITLS_EXE_LINK_FLAGS="-pie;-Wl,-z,now"` |
| `HITLS_BUILD_GEN_INFO` | 生成构建信息文件（macros.txt、sources.txt、include_dirs.txt） | `cmake .. -DHITLS_BUILD_GEN_INFO=ON` |
| `HITLS_PLATFORM_ENDIAN` | 手动指定字节序（通常无需手动设置） | `cmake .. -DHITLS_PLATFORM_ENDIAN=little/big` |
| `HITLS_PLATFORM_BITS` | 手动指定平台位数（通常无需手动设置） | `cmake .. -DHITLS_PLATFORM_BITS=64/32` |
| `HITLS_PLATFORM_INT128` | 手动指定是否支持 128 位整数（默认开启） | `cmake .. -DHITLS_PLATFORM_INT128=OFF` |

#### 3.1.1 常用的配置命令

```bash
# 默认构建（启用所有特性，同时生成静态库和共享库）
cmake ..

# 使用预设——全量构建
cmake .. -DHITLS_BUILD_PROFILE=full

# 开启特定算法特性（不使用预设时）
cmake .. -DHITLS_CRYPTO_SHA256=ON

# 关闭特定算法特性
cmake .. -DHITLS_CRYPTO_SHA256=OFF

# 仅生成静态库
cmake .. -DHITLS_BUILD_SHARED=OFF

# 仅生成动态库
cmake .. -DHITLS_BUILD_STATIC=OFF

# 打包为单个库
cmake .. -DHITLS_BUNDLE_LIB=ON

# 构建命令行工具
cmake .. -DHITLS_BUILD_EXE=ON

# 启用汇编优化（自动匹配当前平台）
cmake .. -DHITLS_ASM=ON

# 指定 x86_64 汇编优化（全量构建）
cmake .. -DHITLS_BUILD_PROFILE=full -DHITLS_ASM_X8664=ON

# 覆盖编译选项
cmake .. -DHITLS_COMPILE_OPTIONS="-O0;-g"

# 覆盖共享库链接选项
cmake .. -DHITLS_SHARED_LINK_FLAGS="-shared;-Wl,-z,now"

# 覆盖可执行文件链接选项
cmake .. -DHITLS_EXE_LINK_FLAGS="-pie;-Wl,-z,now"

# 追加编译选项
cmake .. -DCMAKE_C_FLAGS="-O0 -g"

# 修改构建类型(CMAKE 会自动添加对应的编译选项，例如 Debug 模式会添加 -g，Release 模式会添加 -O3 等)
cmake .. -DCMAKE_BUILD_TYPE=Debug/Release/RelWithDebInfo/MinSizeRel

# 通过 CMake 的 -C 参数加载自定义配置文件
# 您可以在自定义配置文件中设置任意支持的 CMake 变量，例如：
#     set(HITLS_CRYPTO_SHA256    ON CACHE BOOL "" FORCE)
#     set(HITLS_BUILD_EXE        ON CACHE     BOOL "" FORCE)
#     set(HITLS_COMPILE_OPTIONS  "-O0;-g" CACHE STRING "" FORCE)
#     set(HITLS_PRESET_LOADED    ON CACHE BOOL "" FORCE) # 自定义配置文件内必须包含该行以标识预设已加载
cmake .. -C ../path/to/your_feature_config.cmake
```

#### 3.1.2 交叉编译
1. 使用工具链文件(推荐)
交叉编译 openHiTLS 时，使用 CMake 的 `-DCMAKE_TOOLCHAIN_FILE` 参数传入工具链文件（`cmake/toolchain/` 目录下提供了常见平台的工具链文件），如下：

```bash
cd openHiTLS
mkdir -p ./build
cd ./build
# 使用 aarch64（ARMv8 64 位）工具链文件（需提前安装 aarch64-linux-gnu-gcc）
cmake .. -DCMAKE_TOOLCHAIN_FILE=../cmake/toolchain/aarch64-linux-gnu-gcc.cmake
make -j
```

如需自定义工具链，可参照 `cmake/toolchain/` 下的模板文件编写，然后通过 `-DCMAKE_TOOLCHAIN_FILE` 传入。

2. 直接指定交叉编译器
除了使用工具链文件外，也可以在 CMake 配置阶段直接指定交叉编译器，例如：

```bash
cd openHiTLS
mkdir -p ./build
cd ./build
cmake .. \
    -DCMAKE_SYSTEM_NAME=Linux \
    -DCMAKE_SYSTEM_PROCESSOR=aarch64 \
    -DCMAKE_C_COMPILER=aarch64-linux-gnu-gcc \
    -DCMAKE_CXX_COMPILER=aarch64-linux-gnu-g++ \
    -DCMAKE_ASM_COMPILER=aarch64-linux-gnu-gcc
make -j
```

### 3.2 构建结果安装

安装openHiTLS的构建结果只需要输入如下命令：

```bash
make install
```

头文件默认安装至/usr/local/include，库文件默认安装至/usr/local/lib。若需要自定义安装路径，在cmake配置阶段使用如下命令：

```
cmake -DCMAKE_INSTALL_PREFIX=<自定义路径> ..
```
